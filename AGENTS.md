# AGENTS.md

## README Scope

- Keep `README.md` user-workflow focused.
- Document how end users run `tide` and what outcomes to expect.
- Do not add developer-maintenance instructions to `README.md` (for example: transcript generation, local fixture internals, or contributor-only tooling).
- Put developer-focused workflows in `docs/` instead of `README.md`.

## README PR Requirements

- For every PR that updates `README.md`, append a `## New README` section to the bottom of the PR body.
- The `## New README` section must include the full updated `README.md` contents.
