# tide

`tide` is a deterministic CLI for stacked branch and PR workflows.

The CLI transcript in this README is generated from real command execution. Nothing is mocked: the generator boots a local Gitea server, creates a real repo, and runs the installed `tide` binary from `PATH`.

## Regenerate CLI Transcript

Prerequisites:

- Docker daemon reachable from your shell.
- `tide` installed on `PATH` (for example, `python3 -m pip install -e .`).

Run:

```bash
make readme-transcript
```

What the utility does:

1. Starts a fresh local Gitea container.
2. Creates `tideadmin/tide-readme-demo` via Gitea API.
3. Initializes a real git repo with `origin` pointed at that Gitea server.
4. Executes the scenario with `tide <command> ...` (never `python -m tide...`).
5. Captures output and exit codes and updates the generated block below.

Direct script usage:

```bash
./scripts/readme/regenerate_transcript.py --readme README.md
```

Useful options:

- `--keep-container`: keep Gitea running for debugging.
- `--print-only`: emit generated transcript to stdout without editing README.
- `--workspace /workspace/tmp/tide-readme`: control scratch directory.
